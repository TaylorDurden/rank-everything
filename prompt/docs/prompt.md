# 资产评级平台 - 完整需求与架构 AI 提示词体系

## 一、系统概述与核心目标提示词

### 1.1 愿景声明

```
构建一个智能、通用、可扩展的资产评级平台，让用户能够系统性地评估和优化各种类型的资产，从数字产品到个人技能，从财务健康到人际关系。

核心价值主张：
- **通用性**：支持多种资产类型的统一评估框架
- **智能化**：AI专家系统提供深度分析和个性化建议
- **系统性**：提供完整的评估-分析-改进闭环
- **可操作性**：所有建议都具体、可执行、有时效性
```

### 1.2 目标用户画像

```
主要用户群体：
1. **专业人士**：产品经理、开发者、设计师
   - 需求：专业的产品/技能评估
   - 使用场景：工作优化、职业发展

2. **个人用户**：自我提升者、投资者、学习者
   - 需求：个人资产优化建议
   - 使用场景：个人成长、财务管理

3. **企业用户**：创业公司、中小企业
   - 需求：业务资产评估
   - 使用场景：业务优化、投资决策

4. **教育机构**：培训机构、学校
   - 需求：技能评估和认证
   - 使用场景：教学评估、课程设计
```

## 二、核心功能需求提示词

### 2.1 资产管理系统

```
功能需求：
1. **资产创建**
   - 支持多种预设资产类型（网站、APP、技能等）
   - 自定义资产类型创建
   - 资产基本信息收集（名称、描述、目标等）

2. **资产组织**
   - 标签系统：支持多维度分类
   - 文件夹/集合：资产分组管理
   - 状态追踪：待评估、已评估、存档等

3. **资产关系**
   - 关联资产：建立资产间关联关系
   - 组合资产：多个资产组合评估
   - 继承关系：衍生资产从父资产继承属性
```

### 2.2 智能评估引擎

```
功能需求：
1. **评估方法选择**
   - 手动输入评估表
   - 数据导入（CSV/JSON/API）
   - 自动采集（网站爬取、API集成）
   - 混合模式（结合多种方法）

2. **评估流程**
   - 向导式评估引导
   - 进度追踪和保存
   - 评估数据验证
   - 实时评分预览

3. **评估模板**
   - 预设专业模板
   - 用户自定义模板
   - 模板共享和复用
   - 行业标准模板库
```

### 2.3 AI 专家分析系统

```
功能需求：
1. **专家角色库**
   - 按资产类型匹配专家
   - 多专家协同分析
   - 专家能力可扩展

2. **分析维度**
   - 结构化分析框架
   - 权重可调评估标准
   - 多视角综合分析

3. **智能建议生成**
   - 基于最佳实践的改进建议
   - 个性化行动计划
   - 优先级排序
   - 资源推荐
```

### 2.4 报告与可视化系统

```
功能需求：
1. **报告生成**
   - 多种报告模板（简洁/标准/详细）
   - 可定制报告内容
   - 导出功能（PDF/HTML/Markdown）

2. **数据可视化**
   - 雷达图展示多维度评分
   - 趋势图展示历史变化
   - 对比图展示基准差异
   - 仪表盘展示关键指标

3. **交互式报告**
   - 可点击钻取详细数据
   - 动态筛选和过滤
   - 实时数据更新
```

### 2.5 学习与改进系统

```
功能需求：
1. **改进追踪**
   - 行动计划创建和分配
   - 进度跟踪和提醒
   - 成果验证和记录

2. **基准对比**
   - 行业基准数据
   - 同类资产对比
   - 历史表现对比

3. **学习资源**
   - 相关课程和教程推荐
   - 工具和资源推荐
   - 专家咨询服务对接
```

### 2.6 社交与协作系统

```
功能需求：
1. **分享功能**
   - 公开/私有分享控制
   - 可分享评估报告
   - 可分享改进成果

2. **协作评估**
   - 团队协作评估资产
   - 角色权限管理
   - 协作讨论和评论

3. **社区功能**
   - 评估模板共享
   - 最佳实践案例
   - 专家问答社区
```

## 三、系统架构提示词

### 3.1 整体架构原则

```
架构设计原则：
1. **模块化设计**：各功能模块独立可替换
2. **微服务架构**：核心服务独立部署
3. **事件驱动**：异步处理提高响应速度
4. **可扩展性**：支持水平扩展和功能扩展
5. **安全性**：端到端安全防护
```

### 3.2 核心服务架构

```
服务划分：
1. **网关服务**：请求路由、认证、限流
2. **用户服务**：用户管理、权限控制
3. **资产服务**：资产管理、元数据管理
4. **评估服务**：评估流程、数据收集
5. **AI分析服务**：专家系统、智能分析
6. **报告服务**：报告生成、数据可视化
7. **通知服务**：消息推送、提醒
8. **数据服务**：数据存储、分析
```

### 3.3 数据架构

```
数据层设计：
1. **关系型数据库**：用户数据、资产元数据
2. **文档数据库**：评估数据、报告内容
3. **向量数据库**：AI分析数据、相似性检索
4. **缓存层**：热点数据、会话数据
5. **对象存储**：文件存储、多媒体资源

数据流程：
采集 → 清洗 → 存储 → 分析 → 可视化
```

### 3.4 AI 架构设计

```
AI服务架构：
1. **提示词管理**：专家角色提示词库
2. **模型路由**：根据任务选择合适模型
3. **上下文管理**：会话历史和参考数据
4. **结果结构化**：标准化输出格式
5. **质量监控**：AI输出质量评估
6. **持续优化**：基于反馈优化提示词

支持模型：
- OpenAI GPT系列
- Anthropic Claude系列
- 开源模型（可选）
```

### 3.5 集成架构

```
外部集成：
1. **数据采集集成**
   - 网站分析工具（Google Analytics等）
   - APP分析工具（App Annie等）
   - 社交媒体API
   - 金融数据API

2. **第三方服务**
   - 认证服务（Google Auth、Auth0）
   - 文件存储（AWS S3、Cloudinary）
   - 通知服务（SendGrid、Twilio）
   - 支付服务（Stripe、PayPal）

3. **开发者API**
   - RESTful API接口
   - Webhook支持
   - SDK开发包
```

## 四、非功能需求提示词

### 4.1 性能需求

```
性能指标：
1. **响应时间**
   - 页面加载：< 3秒
   - API响应：< 500ms（95%请求）
   - 报告生成：< 30秒（复杂报告）

2. **并发能力**
   - 支持1000+并发用户
   - AI分析队列管理
   - 批量处理能力

3. **可用性**
   - 系统可用性：99.9%
   - 故障恢复：< 15分钟
   - 数据备份：每日自动备份
```

### 4.2 安全需求

```
安全要求：
1. **数据安全**
   - 端到端加密传输
   - 敏感数据脱敏
   - 数据访问权限控制
   - 数据备份和恢复

2. **应用安全**
   - 防止SQL注入
   - XSS防护
   - CSRF防护
   - 暴力破解防护

3. **合规性**
   - GDPR合规
   - CCPA合规
   - 数据保留策略
   - 用户数据删除权
```

### 4.3 可扩展性需求

```
扩展能力：
1. **水平扩展**
   - 无状态服务设计
   - 负载均衡支持
   - 自动伸缩策略

2. **功能扩展**
   - 插件式架构
   - 新资产类型扩展
   - 新评估方法扩展
   - 新AI专家扩展

3. **集成扩展**
   - 新数据源集成
   - 新第三方服务集成
   - 自定义分析规则
```

### 4.4 用户体验需求

```
用户体验标准：
1. **易用性**
   - 直观的界面设计
   - 清晰的引导流程
   - 智能的默认设置
   - 完善的帮助文档

2. **可访问性**
   - WCAG 2.1 AA标准
   - 屏幕阅读器支持
   - 键盘导航支持
   - 色彩对比度要求

3. **多平台支持**
   - 响应式Web设计
   - 移动端优化
   - PWA支持
   - 未来考虑原生APP
```

## 五、业务逻辑提示词

### 5.1 评估流程业务逻辑

```
核心评估流程：
1. **准备阶段**
   - 资产选择或创建
   - 评估方法选择
   - 数据源配置

2. **数据收集阶段**
   - 数据采集（手动/自动）
   - 数据验证和清洗
   - 数据标准化处理

3. **分析阶段**
   - 基础指标计算
   - AI深度分析
   - 基准对比分析

4. **报告阶段**
   - 报告内容生成
   - 可视化图表生成
   - 建议行动计划生成

5. **跟进阶段**
   - 计划执行跟踪
   - 效果评估
   - 迭代优化
```

### 5.2 AI 分析业务逻辑

```
AI分析流程：
1. **上下文构建**
   - 收集资产相关信息
   - 检索历史评估数据
   - 获取行业基准数据

2. **专家匹配**
   - 根据资产类型选择专家
   - 组合多个专家视角
   - 调整分析深度和范围

3. **分析执行**
   - 结构化分析各个维度
   - 权重调整和计算
   - 生成分析结论

4. **建议生成**
   - 基于分析结果生成建议
   - 优先级排序
   - 资源匹配

5. **质量检查**
   - 逻辑一致性检查
   - 实用性评估
   - 可操作性验证
```

### 5.3 定价与商业模式

```
商业模型设计：
1. **免费层**
   - 基础资产评估（每月5次）
   - 简洁版报告
   - 社区模板使用

2. **专业版**（$29/月）
   - 无限次评估
   - 详细版报告
   - 高级AI分析
   - 自定义模板

3. **团队版**（$99/月）
   - 最多10个用户
   - 团队协作功能
   - 批量评估
   - 数据导出API

4. **企业版**（定制报价）
   - 自定义AI模型
   - 私有化部署
   - 定制评估框架
   - 专属技术支持
```

## 八、风险与应对提示词

### 8.1 技术风险

```
风险点：
1. **AI服务稳定性**
   - 应对：多模型备用、本地缓存、降级方案

2. **数据采集限制**
   - 应对：多种采集方式、用户手动补充、合规设计

3. **性能瓶颈**
   - 应对：异步处理、CDN加速、数据库优化

4. **安全风险**
   - 应对：多层防护、定期安全审计、漏洞赏金
```

### 8.2 业务风险

```
风险点：
1. **用户接受度**
   - 应对：简化流程、教育用户、逐步引导

2. **市场竞争**
   - 应对：差异化功能、社区建设、生态合作

3. **盈利模式**
   - 应对：多种定价策略、企业定制、增值服务

4. **合规风险**
   - 应对：法律咨询、合规设计、透明政策
```
